# Подробный отчет по вариантам деплоя (план)

## Цель

Подготовить читаемый сравнительный отчет по всем основным типам деплоя и популярным сервисам, с фокусом на:

- глобальные варианты (по твоему выбору);
- приоритет простоты настройки;
- отдельное сравнение Docker-подходов и не-Docker PaaS.

## Что войдет в итоговый отчет

- Короткий обзор типов деплоя:
  - PaaS (Render/Railway/Fly/Koyeb/Northflank)
  - Split-хостинг (UI отдельно, API отдельно)
  - VPS + Docker Compose
  - Serverless containers (Cloud Run и аналоги)
- Единая таблица сравнения по сервисам:
  - Docker support
  - free tier / trial
  - sleep/cold start
  - сложность старта
  - БД (managed Postgres / ограничения)
  - когда подходит / когда не подходит
- Отдельный блок «Лучший выбор под твой проект»:
  - быстрый старт
  - бесплатный старт
  - прод-готовый путь

## Метод оценки

- Нормализовать критерии в одинаковую шкалу: простота, стоимость старта, эксплуатационные риски.
- Отделить «бесплатно навсегда» от «trial/credits».
- Явно пометить риски: спящий инстанс, лимиты БД, обязательная карта, vendor lock-in.

## Формат

- Компактный документ с короткими выводами по каждому варианту.
- Одна основная сравнительная таблица + 1 мини-таблица рекомендаций под сценарии (MVP, хобби, прод).
- Финальный вывод в 3 шага: что выбрать сейчас, что улучшить через 1–2 месяца, когда переезжать на VPS/K8s.
